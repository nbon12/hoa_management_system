# Implementation Plan: Architecture Remediation — Proper Target Architecture

**Branch**: `015-architecture-remediation` | **Date**: 2026-07-02 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/015-architecture-remediation/spec.md`

## Summary

Remediate the architecture smells catalogued in `docs/architecture-smells-analysis.md` (F1–F20) as six staged, independently deliverable slices: (P1) make provider-event payment recording atomic and idempotent behind one shared `PaymentRecorder`; (P2) centralize business-error mapping and identity-claim access; (P3) add an environment-level production backstop for test machinery; (P4) generate Angular contract types from the existing OpenAPI document with CI drift detection; (P5) enforce layering rules (no Infrastructure→Features, shared kernel in Domain, provider-SDK confinement) via architecture tests; (P6) split test tiers, split CI from CD, extract a shared IaC core module, and decompose the two payment god components. External behavior is preserved except the deliberately specified changes (uniform error envelope, production backstop).

## Technical Context

**Language/Version**: C# / .NET 9.0 (backend); TypeScript / Angular 17.3 (frontend); HCL for OpenTofu ≥ 1.8 (infra); GitHub Actions YAML (CI)
**Primary Dependencies**: FastEndpoints (+ FastEndpoints.Swagger/NSwag for OpenAPI), EF Core 9 (Npgsql), Stripe.net (confined to `Infrastructure/Payments`), Serilog, Sentry, FluentValidation; Angular 17.3 + ngx-stripe; new dev-time additions: `NetArchTest.Rules` (architecture tests), `openapi-typescript` (types-only client codegen)
**Storage**: PostgreSQL (Neon in deployed envs, Testcontainers locally/CI). **No schema changes**: remediation uses existing tables (`PaymentTransactions`, `LedgerEntries`, `WebhookEventInbox`, `OutboxMessages`); consistency detection is report-only (logs/alerts, no new tables)
**Testing**: xUnit; new container-free `HOAManagementCompany.UnitTests` project (unit + architecture tests); existing `HOAManagementCompany.Tests` keeps Testcontainers integration/perf tiers; Jasmine/Karma + Angular Testing Library, Cypress E2E, Playwright smoke, Storybook (frontend)
**Target Platform**: Linux containers on Google Cloud Run (scale-to-zero); Cloudflare Pages (frontend)
**Project Type**: Web application (existing monorepo: .NET backend + Angular frontend + OpenTofu infra)
**Performance Goals**: No regression in API latency; webhook handler transactions stay short-lived (no external provider calls inside DB transactions); unit-test tier completes < 60 s without containers (SC-006)
**Constraints**: Neon low max-connections (pooling, short-lived DbContext — transactions must not span Stripe calls); behavior preservation everywhere except uniform error envelope + production backstop; at-least-once, possibly out-of-order provider event delivery; each slice independently mergeable with the full suite green (SC-007)
**Scale/Scope**: ~42 backend endpoints, 26 entities, 17 Angular components, 9 CI workflows, 2 IaC modules; touched surface is bounded by the 20 findings — no new product surface area

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **Technology fit**: PASS. All work stays on Angular / .NET FastEndpoints / PostgreSQL-Neon / Cloud Run / Cloudflare / GitHub Actions / Sentry / Serilog. OpenAPI is generated by **FastEndpoints.Swagger (NSwag)** per constitution v2.3.0 (amended 2026-07-03 to name the actual toolchain); Swagger stays dev/Dev-only and disabled in production (verified by P3 backstop tests). Auth is the existing ASP.NET Identity + JWT implementation (pre-existing deviation from the constitution's Auth0 wording; being resolved by a separate constitution-amendment PR — out of scope here, not worsened; FR-008 centralizes claim access without changing the auth provider).
- **HOA tenancy**: PASS. No tenancy semantics change. Per-property serialization (FR-001/FR-004) and the shared identity accessor (FR-008) strengthen the existing property-scope boundary; no cross-tenant queries added.
- **API contracts**: PASS. Error responses converge on the documented `{ code, message }` envelope with the business error's status (FR-006) — a non-breaking tightening (previously some endpoints leaked 500s for business errors). Success shapes, pagination (`limit`/`offset`), UTC timestamps, and ID formats unchanged. Client types regenerate from the published OpenAPI description (FR-011); a drift gate prevents contract divergence.
- **Security and operations**: PASS. Secrets handling unchanged; FR-009 adds fail-fast startup validation (FluentValidation, environment-aware — extends the existing 008 validators) rejecting seed/dev-tools flags in Production, and logs refused test-machinery invocations as security-relevant Serilog events. Global exception handler keeps production responses generic; Sentry tracing preserved through the new central error mapping.
- **File storage**: PASS (not applicable beyond removal). FR-010 removes `testdata/sample.pdf` from publish output; R2/MinIO architecture untouched (embedded fallback in `TestDataFiles` already covers local/test needs).
- **Caching/edge**: PASS — no caching changes; error responses remain uncacheable.
- **Testing discipline**: PASS. Test-first red→green for each slice: interrupt-and-retry business-process tests (P1) on PostgreSQL/Testcontainers with per-test isolation; xUnit Theories across the five provider-event types; architecture tests (P5) and pure unit tests move to the new container-free project; per-domain factories reduce `TestDataSeeder` coupling (FR-017). Frontend keeps Jasmine/Karma, Angular Testing Library, Cypress, Playwright, Storybook.
- **CI/CD and documentation**: PASS. Sonar, Codecov, and 95% coverage gates continue to run in the split PR-verification workflow (FR-018); deploy path moves to a separate release workflow triggered on main. Repowise marker regions refreshed in every touched hotspot file (see Repowise section).
- **Executable & living specs**: PASS. Each acceptance scenario maps to a named test class/check (see spec Constitution Requirements); spec 006 (payments) external behavior is preserved — structure-only moves — and will be re-verified against its `spec.md` before each payments PR merges; any drift found is reconciled per constitution §11.

**Gate result: PASS** (no violations requiring Complexity Tracking; the NSwag toolchain is constitution-conformant as of v2.3.0, and the one remaining pre-existing deviation — Identity-vs-Auth0 — is documented above as out of scope, unchanged, and owned by a separate amendment PR).
